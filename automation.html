Behind the Scenes

棱镜PLSM 的 AI 内容流水线

AI 主笔 · 人类把关 · 半自动流水线

这个公众号是 AI 在运营

「棱镜PLSM」公众号的每篇文章，是 openclaw 小龙虾 这套 AI agent 系统的产出。openclaw 调 Brave Search 抓今日热点，AI 负责调研、写作、出图，脚本负责排版和上传。

我作为人类的工作很轻量：决定要不要发，在最终稿上做轻微的口吻调整。剩下的繁琐工序——选题、调研、写作、配图、HTML 排版、上传草稿——全部交给流水线。

📢 透明披露：选题与正文是 AI 写的，配图由 GPT 生成，我只负责最后过审。

技术栈

整套系统跑在我自己的电脑上，不依赖任何商业 SaaS。

openclaw 小龙虾

本地运行的 AI agent 编排工具，CLI 形态，自带 workspace、agent、gateway、auth、hooks。整个流水线的大脑。

DeepSeek-Chat

openclaw 默认接入 DeepSeek 的最便宜模型 deepseek-chat（约 0.2 元 / 百万 input token，缓存命中后更便宜）。负责文章主体写作。

Brave Search

openclaw 推荐配套的搜索后端。写作 agent 在动笔前调 Brave 抓相关背景资料、事实数据、近期报道，避免幻觉。

写作 Agent

针对公众号阅读习惯调过的提示词：碎碎念排版、每段不超 3 句话、强情感共鸣、可读性优先。出稿即可发。

GPT 生图 + 微信草稿 API

配图调 GPT 出 900×500 公众号封面；Python 脚本调微信公众平台官方 API：上传封面拿 media_id、HTML 格式化文章、推送草稿箱。

工作流程

从「Brave 抓回今日热搜」到「公众号里多一条草稿」，前后不到 90 秒。

1 · 抓热点

openclaw 调 Brave Search 拉今日热搜与新闻，按权重打分，AI 自动筛出最契合公众号定位（时事观察 / 人文思考）的 1 条。

2 · AI 调研

openclaw 调 Brave Search 抓相关背景资料、事实数据、近期报道，整理成上下文喂给写作 agent。

3 · 写作

DeepSeek-Chat 按 brief + 风格指南写 800 字左右的成文，自动加碎碎念排版（每段一两句、大量空行）。

4 · GPT 生图

调 GPT 生图（DALL-E / gpt-image），按文章主题出 900×500 微信公众号标准封面。

5 · 自动上传

脚本调微信草稿 API：上传封面 → 拿 media_id → 推送 HTML 文章 → 落到草稿箱。

6 · 人类审核

我打开公众号后台扫一眼，调整一两个用词，点发布。整个手工环节通常不超过 2 分钟。

openclaw 配置一瞥

实际跑这套流水线的 ~/.openclaw/openclaw.json 节选：

{
  "models": {
    "providers": {
      "deepseek": {
        "baseUrl": "https://api.deepseek.com/v1",
        "models": [
          { "id": "deepseek-chat", "contextWindow": 200000 }   // 最便宜的写作模型
        ]
      }
    }
  },
  "search": {
    "provider": "brave",                          // openclaw 推荐
    "apiKey":   "${BRAVE_API_KEY}"
  },
  "agents": {
    "defaults": {
      "model": { "primary": "deepseek/deepseek-chat" },
      "workspace": "~/.openclaw/workspace"
    }
  },
  "gateway": { "port": 18789, "mode": "local" }
  // 真实 token / API key 已隐去
}

为什么这么做

传统公众号运营成本高、周期长。一个人撑不住每天更新；找团队又烧钱。

我把它当成一个工程问题：用 AI agent 编排把「选题 + 写作 + 排版 + 发布」做成可重复的流水线。人类只在最后做「质量阀门」。

结果就是：一个人 + 一台电脑 + 每月不到 5 块钱的 API 费用，把繁琐的工序全部自动化掉，省下来的时间花在选题质量和最终把关上。
